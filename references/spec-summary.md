## 总体职责与流程

- 技术支撑部门负责制定制度、参与符合性评审、监督规范执行。
- 业务部门负责明确需求、参与设计评审、组织 UAT、验收、试运行并评价软件质量。
- 建设/开发团队负责确保开发符合规范，组织代码审查、设计评审、安全测试，对不符合项整改至满足要求。
- 开发过程应覆盖前端交互、软件编码、软件设计、质量安全、大屏设计等分册要求。

## 后端 Java 编码

- **命名**：类名使用名词/名词短语并采用 PascalCase；方法、变量、参数使用有业务含义的 camelCase；避免拼音缩写、匈牙利命名、无意义缩写、仅大小写不同的标识符。
- **结构**：源文件按 package、import、类说明、成员属性、方法顺序组织；禁止通配符 import；import 应清晰、按规则分组；properties 等配置文件使用 UTF-8。
- **类与接口**：单个类保持职责单一；接口命名遵循 Java 规范，不随意加不必要后缀；抽象、实现、DTO、VO、Entity、Mapper/DAO、Service、Controller 分层清晰。
- **语句块**：复杂条件建议使用括号明确优先级；不要为了技巧牺牲可读性；if/else、switch、循环保持结构完整、易读，避免深层嵌套。
- **异常**：异常处理应兼顾封装性和可追踪性；不得吞异常；不得向用户暴露堆栈、SQL、路径、密钥等敏感调试信息。
- **线程池**：需要并发时优先使用受控线程池；避免直接频繁创建线程；线程池参数、拒绝策略、超时与关闭行为应可解释。
- **资源释放**：数据库连接、流、文件、网络资源必须可靠关闭，Java 优先使用 try-with-resources。
- **性能**：大批量字符串拼接使用 `StringBuilder`/`StringBuffer`；集合转数组和循环处理避免不必要复制；减少多层循环和重复数据库访问。
- **安全**：禁止拼接 SQL；禁止把未经校验的数据输出到页面；敏感信息不得明文存储、硬编码或写入日志。
- **日志**：日志用于审计、排障和运行监控；应记录权限、账号、登录认证、系统自身、交易流水、错误等关键事件；日志内容包含时间、主体、操作、结果、关键业务标识，避免记录密码、令牌、完整证件号等敏感信息。

## 前端 JavaScript / Vue / Uni-app

- **编码与格式**：文件使用 UTF-8；命名清晰一致；目录按资源类型和业务模块划分；保持统一缩进、空格、换行和语句风格。
- **语言使用**：避免隐式全局变量；优先使用严格相等；避免复杂隐式类型转换；循环、局部变量、字符串连接、对象创建应兼顾性能。
- **注释**：复杂业务、分支、循环和非显然逻辑需要注释；注释解释意图，不重复代码字面含义。
- **安全**：用户输入必须校验；动态 HTML/富文本必须净化或编码；避免直接使用不可信内容写入 `innerHTML`；上传下载需限制类型、大小、路径和权限。
- **交互**：新增、修改、删除、查询等常用流程应有明确按钮、确认、结果反馈和错误提示；危险操作必须有二次确认或安全提示。
- **移动端/Uni-app**：遵循组件库接入规范，统一组件来源、初始化、样式和目录组织；不得私自引入不受控组件造成风格和安全不一致。

## 软件设计与微服务

- **总体设计**：设计文档应明确业务边界、模块划分、数据流、接口、异常、日志、安全、性能、部署和兼容性要求。
- **分层架构**：Controller 只负责请求适配和参数校验；Service 处理业务编排和事务；DAO/Mapper 负责持久化；实体、DTO、VO 分离。
- **接口设计**：接口路径、方法、参数、返回结构、错误码、分页、排序、权限和幂等性应清晰；接口注解/文档应能支持自动生成或评审。
- **服务调用**：微服务间调用应有超时、重试、熔断、降级和异常兜底；不得无限重试或阻塞主链路。
- **网关认证**：通过 Token 等机制进行统一认证；网关白名单需最小化、可追踪、定期复核；不得将敏感管理接口加入宽泛白名单。
- **配置**：环境相关参数进入配置中心或配置文件；不得把账号、密码、密钥、令牌硬编码在源码中。
- **单元测试**：核心业务、边界条件、异常路径和安全逻辑应具备测试；测试命名和结构应可读。

## 数据库与 SQL

- **命名**：库/表空间、表、字段、索引、视图、触发器、函数/存储过程命名应统一、可读、体现业务含义；避免无意义缩写和仅大小写差异。
- **表设计**：表应有主键；字段类型、长度、默认值、是否为空、注释和约束应明确；重要字段保留创建/更新人、时间、删除标识等审计信息时应保持一致。
- **索引**：按查询条件、关联字段、排序字段设计索引；避免过多无效索引；大表变更需评估锁表与性能影响。
- **查询**：禁止 `SELECT *` 进入生产关键代码；大结果集必须分页；避免循环内逐条查询造成 N+1；避免函数包裹索引列导致索引失效。
- **事务**：事务范围保持最小；避免在事务内执行耗时外部调用、文件操作或非必要日志写入；并发更新按固定顺序访问数据以减少死锁。
- **安全**：SQL 使用参数化绑定或 ORM 安全 API；禁止字符串拼接用户输入；敏感字段脱敏展示、加密/哈希存储。

## 质量与安全

- **代码质量评价**：关注规范性、可读性、可维护性、复杂度、重复率、可靠性、安全性、性能和测试覆盖。
- **Web 部署**：生产环境关闭调试模式；限制目录浏览；合理配置安全响应头、HTTPS、Cookie 安全属性和跨域策略。
- **身份验证**：密码策略、登录失败限制、验证码/ MFA（按场景）、会话超时和退出失效机制应明确。
- **权限管理**：服务端强制鉴权；前端隐藏按钮不能替代后端权限校验；越权访问、水平越权、垂直越权必须防范。
- **敏感数据保护**：识别个人信息、账号、密码、密钥、令牌、财务等敏感数据；传输加密、存储加密/哈希、展示脱敏、日志去敏。
- **输入校验**：所有外部输入按白名单、类型、长度、范围、格式校验；后端校验不可省略。
- **输出编码**：按 HTML、JS、CSS、URL、SQL、Shell 等上下文正确编码，防止 XSS、注入和命令执行。
- **上传下载**：限制文件类型、大小、扩展名、MIME、存储路径；文件名随机化；防止路径穿越；下载鉴权并校验资源归属。
- **异常处理**：统一错误码和错误响应；用户侧提示友好，服务端保留可追踪日志；禁止泄漏内部实现。
- **密码规范**：系统密码和应用密码不得明文存储；使用强哈希/加盐或安全密钥管理；默认密码必须强制修改。
- **SDL**：培训、需求、设计、实现、验证、发布、响应阶段均应有安全活动和检查清单；需求阶段识别敏感数据和风险，设计阶段做威胁建模，实现阶段做安全编码与代码扫描，验证阶段做安全测试，发布阶段确认风险闭环，响应阶段跟踪漏洞修复。

## UI/UE 与前端交互

- **一致性**：页面布局、字体、颜色、按钮、表单、提示语、字段名称、流程动作保持一致。
- **页面设计**：标题清晰；页面结构符合用户流程；常用操作可见；重要操作突出；错误、成功、加载、空数据状态必须有反馈。
- **控件**：查询、新增、修改、删除等控件遵循统一位置和交互；删除/提交等风险操作增加确认；表单校验即时、明确。
- **通用格式**：日期、时间、金额、数量、百分比、地区、单位、编号等格式统一；同一字段在不同页面命名一致。
- **媒体命名**：图片、图标、音视频等资源命名应表达用途并遵循目录分类，避免中文空格和无意义名称。

## 大屏设计

- **适用范围**：公司内部大屏展示产品；非大屏系统遵循前端交互分册。
- **设计原则**：保证设计一致性，提升开发效率，辅助理解业务，方便迭代。
- **跳转**：页面尽量少跳转，建议最多 3 次；必须跳转时提供返回入口，形成闭合循环；大屏跳转需平滑高效，避免卡死或逻辑异常。
- **分辨率**：PC、大屏、触摸屏设计以 `1920×1080` 或 `3840×2160` 为基准；拼接屏可按 `32:9`、`3840×1080`；适配常见分辨率，避免错位。
- **标识**：主体页面必须使用中国铁塔 logo；可在标识右侧增加系统名称；避免无关内容干扰。
- **文字**：文字必须完整展示，不得被遮挡或重叠；过长时可省略并用悬浮提示展示详情；标题建议加粗且字号大于正文。
- **数据**：敏感数据应具备分权分域能力；指标说明清晰；计量单位一致；大数据列表需支持分页、排序、下载、打印等按需能力。
- **间距与布局**：推荐 24 列自适应栅格；根据关联性组织间距；建议行间距约 `24px`（14px 字体）到 `30px`（16px 字体）。
- **配色**：背景色饱和度建议在 5–12；图表内容明度建议在 55–80；同类状态颜色保持一致。
- **组件**：列表、分页、按钮、下拉、时间/地区选择、返回、滚动条等组件统一；同一字段命名和下拉选项顺序保持一致。
- **图表**：图表应有清晰标题；标题、标签、轴标签、提示同类字体字号一致；柱状图、折线图等建议有平均值线；钻取、关联、下钻、导出需权限控制。

## 自动扫描覆盖点

`scripts/tower_compliance_scan.py` 能快速发现以下高风险线索：

- 明文密码、密钥、令牌、数据库连接串。
- Java/SQL 中疑似拼接 SQL、`${}` SQL 注入风险、`SELECT *`。
- 前端疑似 XSS：`innerHTML`、`v-html`、`dangerouslySetInnerHTML` 等。
- 异常吞噬、`printStackTrace`、`System.out.println`、调试输出。
- 硬编码 IP/URL、HTTP 明文地址、宽泛 CORS。
- 未限制上传类型/大小的常见上传入口线索。
- 大文件或前端资源中不符合目录/命名习惯的线索。

扫描器不能替代人工评审。对 UI/UE、大屏视觉、架构设计、权限模型、事务边界、索引有效性、SDL 流程等必须结合上下文人工检查。
